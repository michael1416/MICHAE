QuE es un Dato?

Un dato es un valor individual, sin contexto, que representa una unidad básica de información. Los datos pueden ser números, palabras, imágenes, sonidos, etc., que por sí solos no tienen un significado completo hasta que se procesan y se les da contexto.

¿Qué es información?

La información es el conjunto de datos procesados y organizados de tal manera que tienen significado y relevancia. La información se utiliza para la toma de decisiones y la comprensión de un contexto específico.

Diferencia entre dato e información

Dato: Es crudo, sin procesar, aislado y carece de significado en su forma original. Información: Es el resultado del procesamiento y organización de los datos, lo que les otorga contexto y significado útil.

Tipos de datos

Datos numéricos: Enteros (integer), decimales (float). Datos de texto: Cadenas de caracteres (strings). Datos booleanos: Verdadero o falso (true/false). Datos de fecha y hora: Fecha y hora (datetime). Datos binarios: Archivos o datos codificados en binario (imágenes, archivos de audio).

¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear un modelo para organizar y estructurar los datos de una manera que represente adecuadamente la información y las relaciones entre los datos en un sistema de información. Este proceso ayuda a definir la estructura lógica y física de la base de datos.

Tipos de bases de Datos, según:

Variabilidad Estáticas: Datos que no cambian o cambian muy raramente. Dinámicas: Datos que cambian con frecuencia. Contenido Bibliográficas: Contienen referencias a otros documentos. Texto completo: Contienen el texto completo de los documentos. Numéricas: Contienen datos numéricos. Multimedia: Contienen imágenes, audio, vídeo, etc. Modelo de base de datos Jerárquicas: Datos organizados en una estructura de árbol. En red: Datos organizados en una estructura de gráfico. Relacionales: Datos organizados en tablas relacionadas entre sí. Orientadas a objetos: Datos organizados como objetos. Documentales: Datos almacenados en documentos (ej. JSON, XML). Grafos: Datos organizados como nodos y relaciones entre ellos.

¿Qué es el modelo relacional?

El modelo relacional es un tipo de modelo de base de datos donde los datos se organizan en tablas (relaciones) que están relacionadas entre sí mediante llaves primarias y foráneas. Cada tabla consiste en filas (tuplas) y columnas (atributos), y cada fila representa una entidad única.

Describa los elementos del Diagrama entidad Relacional

Entidad: Representa objetos o cosas del mundo real. Se muestra como un rectángulo. Atributo: Características o propiedades de una entidad. Se muestra como un óvalo. Relación: Asocia dos o más entidades. Se muestra como un rombo. Llave primaria: Identificador único para cada entidad. Llave foránea: Atributo que crea una relación entre dos tablas.

¿Qué es una entidad?

Una entidad es un objeto del mundo real que tiene una existencia independiente y puede ser identificable de manera única en un sistema. Por ejemplo, un estudiante, un libro, o un curso.

¿Qué es un atributo y sus tipos

Un atributo es una propiedad o característica de una entidad. Tipos de atributos incluyen:

Simples: No se pueden dividir (ej. número de identificación).

Compuestos: Se pueden dividir en sub-partes (ej. dirección: calle, ciudad, código postal).

Derivados: Calculados a partir de otros atributos (ej. edad a partir de la fecha de nacimiento).

Multivaluados: Pueden tener múltiples valores (ej. números de teléfono)

¿Qué es una relación?

Una relación es una asociación entre dos o más entidades. Define cómo las entidades están conectadas entre sí.

¿Qué son las llaves primarias, foráneas y sus características?

Llave primaria: Un atributo o conjunto de atributos que identifican de manera única una fila en una tabla. Debe ser único y no nulo. Llave foránea: Un atributo en una tabla que crea un vínculo con la llave primaria de otra tabla. Permite mantener la integridad referencial entre las tablas.

¿Qué son las reglas del Negocio?

Las reglas del negocio son condiciones, restricciones y políticas que definen cómo se deben gestionar y manipular los datos en un sistema. Estas reglas aseguran que las operaciones de la base de datos reflejen correctamente los procesos y políticas de la organización.

¿Qué es la cardinalidad o mapeo?

La cardinalidad se refiere a la naturaleza de la relación entre dos entidades, indicando el número de instancias de una entidad que pueden asociarse con una instancia de otra entidad. Tipos de cardinalidad incluyen: Uno a uno (1:1): Una entidad A se relaciona con una sola entidad B. Uno a muchos (1 ): Una entidad A se relaciona con muchas entidades B. Muchos a muchos (M ): Muchas entidades A se relacionan con muchas entidades B. Estos conceptos forman la base de la organización y gestión de datos en sistemas de información y bases de datos.
